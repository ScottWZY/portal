## 一、任务概述

### 目标
为 VitePress 面试准备 Wiki 项目的面试冲刺（interview）模块，从当前仅有的 1 个概览页，扩展为 **10 个完整页面**，覆盖面试策略、算法刷题、系统设计、行为面试、简历优化、模拟面试、软技能、知识串联等全部面试准备维度。

### 背景
当前 `interview/` 模块已有：
- ✅ `index.md` — 模块概览（已完成，~120 行）

当前问题：
1. **侧边栏规划了 4 个页面，但均未创建**：`algorithm/index.md`、`algorithm/hot-questions.md`、`system-design/index.md`、`behavioral/index.md`
2. **index.md 覆盖了大量内容维度，但缺少对应子页面**：面试策略、知识体系串联、高频真题速查、简历优化、软技能、模拟面试等板块在 index.md 中已有内容框架，但无独立页面深入展开
3. **与 16 周计划（plan.md）第 13-16 周的内容对应不完整**：第 15 周（简历优化/项目复盘）、第 16 周（模拟面试/最终冲刺）的内容缺乏独立页面支撑

### 约束条件
- 使用 VitePress 兼容的 Markdown 格式（支持 mermaid 图表）
- 每个页面字数约 2000–4000 字
- 内容面向高级工程师面试，需有深度和实战性
- 所有内容使用中文编写
- 文件仅操作项目路径内的文件（`wiki-docs/docs/interview/`）
- 每页末尾至少包含 5 道经典高频面试题 + 参考答案

## 五、格式规范

### 页面结构模板
```markdown

> 创建日期：YYYY-MM-DD

|--------|----------|----------|----------|

...

...

## 三、代码示例 / 实战案例
...

## 四、常见误区与坑点
...

## 五、面试高频问题汇总

### Q1：题目
**参考答案：**
...

（至少 5 题）
```

### VitePress 容器语法
- `::: tip 标题` / `::: warning 标题` / `::: danger 标题` / `::: info 标题` / `::: details 标题`

### 内容要求
- 每个页面末尾至少 5 道面试题 + 参考答案
- 关键流程使用 mermaid 图表
- 包含方案对比表格
- 关键代码用 Java 伪代码 / 核心片段
- 页面间交叉引用，形成知识网络
