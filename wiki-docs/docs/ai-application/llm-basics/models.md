## 六、关键趋势观察

### 6.1 API 兼容标准化

现在 **几乎所有模型都兼容 OpenAI SDK**，只需替换 `base_url` 和 `api_key` 即可切换模型，迁移成本极低。

```python
# 统一调用代码，切换模型只需改两行
client = OpenAI(
    api_key="YOUR_API_KEY",
    base_url="https://api.xxx.com/v1"  # 切换模型服务商
)
```

### 6.2 价格战持续

DeepSeek V4-Pro 永久降价 75%，模型价格持续下探。现在最便宜的输入价格已降到 **$0.10/M tokens** 以下，比一年前便宜了一个数量级。

### 6.3 推理模型分化

o3/o4-mini（OpenAI）、DeepSeek R1（深度求索）、Claude Extended Thinking（Anthropic）等推理专用模型形成独立赛道。复杂推理任务交给专用推理模型，成本和准确率都更好。

## 七、面试高频题

### Q1: GPT-4o 和 Claude 系列如何选择？它们的核心差异在哪里？

**详细答案：**
GPT-4o（OpenAI）和 Claude（Anthropic）是当前业界两大旗舰模型系列，它们在设计理念和能力侧重上有明显区别。GPT-4o 的核心优势是多模态理解能力更强（图像、视频输入）、OpenAI 生态成熟（Function Calling、Code Interpreter 等工具链完整）、对英文世界知识覆盖更全面。而 Claude 的优势在于代码能力更强（SWE-bench 得分更高）、上下文窗口更大（全系标配 1M）、长文档处理更稳定、更擅长复杂 Agent 任务和 Computer Use。

选型决策框架：如果你的场景是**Agent 编程、复杂代码重构、超长文档分析**，优先选 Claude Sonnet 或 Opus；如果你的场景是**多模态图像/视频理解、需要成熟生态工具、英文内容生成**，优先选 GPT-4o。从成本角度看，Claude Sonnet 4.6 定价 $3/$15 每百万 tokens（输入/输出），GPT-4o 定价 $2.5/$10，价格差异不大，但 Claude 在同样价格下提供更大上下文窗口。实际项目中一个常见做法是核心任务用 Claude，非核心批量任务用 Gemini Flash 或 DeepSeek 来降本。

面试中一个加分项是提到：Anthropic 在 RLHF 阶段更注重"无害性"对齐，Claude 的输出通常更符合人类价值观，拒绝有害请求的概率更高，但有时也会因为过度谨慎而拒绝一些完全合理的请求。OpenAI 的对齐相对宽松，对边界请求的接受度更高。

### Q2: 国内模型和国际模型的主要差异是什么？什么情况下应该选择国内模型？

**详细答案：**
国内模型（DeepSeek、Qwen、GLM、Kimi 等）和国际模型（GPT、Claude、Gemini）主要差异体现在三个方面。第一，**中文能力**：国内模型在中文语料上训练更充分，对中文口语、网络梗、国内常识理解更好，Qwen3-Max 的中文理解已超越 GPT-4o。第二，**合规性**：国内模型服务商通常提供合规备案，满足企业对数据不出境的要求，适合政府和国企项目。第三，**价格**：国内模型 API 价格普遍比国际模型低 50%~80%，DeepSeek V3 输入价格仅 $0.27/M，是 GPT-4o 的十分之一左右。

选择国内模型的典型场景有四个：第一，**要求数据合规，数据不能出境**——国内部署或国内 API 服务商满足监管要求；第二，**中文为主的业务场景**——比如中文客服、中文文档处理，国内模型性价比更高；第三，**高并发、低延迟要求**——国内节点网络延迟比调用海外 API 低一个数量级（几十毫秒 vs 几百毫秒）；第四，**大规模批量处理**——成本敏感场景用国内模型可以大幅降低账单。

但也要认识到局限性：国际模型在复杂推理、多模态、英文世界知识上仍然领先。如果你的业务是面向全球、需要处理大量英文内容、或依赖成熟工具生态，国际模型仍然是更好选择。实际工程中，混合架构很常见：核心推理用 GPT/Claude，批量文本预处理用国内 Flash 模型降级降本。

### Q3: 如何根据业务场景选择合适的大模型？请给出几个具体例子。

**详细答案：**
模型选型本质是在"能力、成本、延迟、合规"四个维度做 trade-off，没有绝对的"最好"模型，只有最适合场景的模型。通用决策框架是：先明确场景的核心约束（成本敏感？必须长上下文？需要合规？），再在满足约束的前提下选择能力最强的模型。

几个典型场景的具体选型：

**Agent/自主编程场景**：推荐 Claude Opus 4.6 或 Sonnet 4.6，因为 Claude 在 SWE-bench 得分领先，对复杂工具调用和多步推理更稳定，Computer Use 生态成熟。如果预算有限，DeepSeek V4-Pro 是性价比极高的替代方案。

**生产环境高并发客服场景**：推荐 Gemini 2.5 Flash 或 DeepSeek V3，它们价格低、速度快、上下文窗口足够（1M），满足大部分通用对话需求，单并发成本比旗舰模型低一个数量级。

**复杂数学/学术推理场景**：推荐 OpenAI o3 或 DeepSeek R1，这些是专门针对推理优化的模型，原生支持长推理链，准确率比通用模型高出很多。如果成本敏感，o4-mini 性价比很高。

**超长文档处理（>100K tokens）**：推荐 Claude Sonnet 或 Gemini 2.5 Pro，都支持 1M 上下文窗口，长上下文质量相对稳定。国内场景选 Qwen3.5-Plus 也支持 1M 窗口。

**极致成本控制批量处理**：选 Gemini 2.5 Flash-Lite 或 Qwen-Flash，输入价格低至 $0.10/M 以下，1M 窗口，适合大规模文本 embedding、分类、摘要等非核心任务。

关键原则是：**不要一开始就上最贵的旗舰模型**，先解决问题，再根据瓶颈升级。很多团队上来就用 GPT-4o 处理所有任务，最后账单上天，其实 80% 的任务用 Flash 级别模型就能搞定。

### Q4: 价格因素如何影响模型选型决策？什么情况下应该追求极致性价比？

**详细答案：**
大模型 API 定价差异非常大，从 $0.10/M 到 $10/M 输入价格，差了 100 倍。理解定价结构是控制成本的第一步：几乎所有服务商都采用"输入+输出分开计费"，且**输出价格通常是输入价格的 2~4 倍**。某些模型还提供前缀缓存优惠，相同的 system prompt 重复调用时，缓存命中部分价格可以降到原价的十分之一（如 DeepSeek 缓存命中只需 $0.028/M）。

追求极致性价比的典型场景：第一，**大规模批量处理任务**——比如对百万级文档做预处理、摘要、分类、embedding，单次任务成本不敏感，但量级上去后总成本差异巨大；第二，**高并发生产环境**——日调用量百万次以上，即使单个请求便宜几分钱，累计下来也是可观的节省；第三，**内部工具和非核心流程**——对准确率要求不是极致，能用就行，性价比比顶尖能力更重要；第四，**原型验证阶段**——快速迭代验证想法，不需要用旗舰模型烧钱。

工程实践中，**模型降级策略**是控制成本的核心。就是把任务分层：简单任务（摘要、分类、客服对话）用 Flash/Mini 模型，中等难度任务用中端模型，只有核心复杂推理任务才用旗舰模型。据业内统计，通过合理的模型降级，在能力损失可接受的情况下，整体 API 成本可以降低 60%~80%。另一个技巧是善用缓存：相同前缀（如 system prompt、few-shot examples）尽量保持不变，让服务商缓存命中，既降价又提速。

### Q5: 为什么现在所有模型都兼容 OpenAI API 格式？这带来了什么好处？

**详细答案：**
OpenAI API 格式已经成为事实上的行业标准，原因在于它先占优势+设计简洁。OpenAI 是第一个做大模型商业 API 的厂商，早期开发者都基于它的 SDK 开发，形成了生态锁定。后来的服务商发现，如果自己不兼容，开发者迁移成本很高，就不愿意用你的服务。所以大家都选择兼容，开发者只需要改两行代码（base_url + api_key）就能切换模型，对服务商来说获客更容易。

这带来的好处对各方都是共赢。对**开发者**来说：不需要为每个模型服务商学习一套新的 SDK，不需要改大量代码就能切换模型供应商，避免了厂商锁定。对**模型服务商**来说：可以直接复用 OpenAI 生态的大量工具和客户端代码，不需要教育市场，用户上手快。对**整个生态**来说：标准化促进了竞争，用户可以随时用脚投票，倒逼各家服务商在价格和质量上竞争，最终推动行业整体进步。

一个常见的误区是认为"兼容 OpenAI 格式意味着模型能力和 OpenAI 一样"——格式兼容只是接口层面，模型能力还是要看训练数据和模型规模。但接口标准化确实极大地降低了切换成本，现在很多团队在生产环境做多模型 fallback：主模型用 A，如果 A 挂了或超时，自动切到 B，对业务完全透明。如果没有标准化，这种熔断降级实现起来会复杂很多。
