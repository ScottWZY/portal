# 前端面试考点概览

## ⭐ 面试重点速览

| 考察维度 | 重点内容 | 大厂权重 | 准备优先级 |
|----------|----------|----------|------------|

## 模块概述

前端面试已经从"会用框架"时代进入"理解原理 + 系统设计 + 工程化思维"的时代。本模块为**高级前端面试准备**提供系统化的知识体系，涵盖手写题、框架原理、算法真题、系统设计和热点问答五大核心领域。

::: danger 2025-2026 面试趋势
- **手写题比重持续上升**：大厂一面几乎必考手写题（Promise.all / 深拷贝 / 防抖节流）
- **框架深度要求提高**：不再满足于"用过"Vue3/React19，要求能说出原理和设计取舍
- **系统设计成为高频考点**：虚拟列表、状态管理库、路由系统设计
- **算法难度向中等靠拢**：链表/树/动态规划成为标配
- **TypeScript 成为默认要求**：大厂前端岗位 90% 以上要求 TS 能力
:::

## 面试考察六大维度

### 一、JavaScript 基础（占比 40%~50%）

| 子领域 | 核心考点 | 典型考题 |
|--------|----------|----------|
| 原型链 | `__proto__` vs `prototype`、继承方式、`instanceof` 原理 | 手写 new / instanceof |
| 闭包 | 词法作用域、内存泄漏、实际应用 | 循环 + setTimeout 输出题 |
| this 指向 | 四条绑定规则、箭头函数、call/apply/bind | 手写 call/apply/bind |
| Event Loop | 宏任务/微任务、Promise 时序、async-await | 输出顺序题 |
| 深浅拷贝 | JSON 方案局限性、循环引用、特殊对象 | 手写深拷贝 |
| Promise | 状态机、链式调用、并发控制 | 手写 Promise.all / race |
| 模块化 | ESM vs CommonJS、Tree Shaking | 循环引用处理 |

### 二、框架深度（占比 20%~30%）

| 框架 | 核心考点 | 面试频率 |
|------|----------|----------|
| Vue3 | 响应式原理（Proxy）、编译优化（PatchFlag/Block Tree/静态提升）、Composition API 设计理念、keep-alive 原理、nextTick 原理 | 极高 |
| React | Fiber 架构、Hooks 原理与限制、React 19 新特性、并发模式、RSC、受控/非受控 | 极高 |
| 对比 | Vue vs React 设计哲学差异、适用场景选择 | 高 |

### 三、工程化能力（占比 10%~15%）

| 子领域 | 核心考点 |
|--------|----------|
| 构建工具 | Vite vs Webpack、ESBuild/Rolldown、Tree Shaking、Code Splitting |
| 性能优化 | 首屏优化、懒加载、虚拟列表、CDN 策略、缓存策略 |
| 代码规范 | ESLint/Prettier、Husky、Commitlint |
| CI/CD | 自动化构建、部署流水线、灰度发布 |
| 微前端 | qiankun/Micro-app/Module Federation、样式隔离、JS 沙箱 |

### 四、算法与数据结构（占比 5%~10%）

| 难度 | 典型题目 | 考察点 |
|------|----------|--------|
| 简单 | 有效括号、反转链表、合并有序数组 | 栈/链表基础操作 |
| 中等 | LRU 缓存、最长不重复子串、三数之和 | 设计 + 算法结合 |
| 较难 | 二叉树遍历（迭代）、买卖股票系列 | 树/DP 熟练度 |

::: tip 前端算法特点
前端算法通常不考太难的（如红黑树、KMP），重点考察**工程思维 + 算法基础**的结合，如 LRU 缓存就是典型的前端算法题。
:::

### 五、系统设计（占比 5%~10%）

| 设计题目 | 考察能力 |
|----------|----------|
| 图片懒加载组件 | 组件 API 设计、IntersectionObserver、边界处理 |
| 无限滚动/虚拟列表 | 性能意识、DOM 复用、滚动计算 |
| 前端路由系统 | hash vs history、路由匹配、导航守卫 |
| 简易状态管理库 | 发布-订阅模式、响应式原理、插件机制 |

### 六、项目经验（贯穿整个面试）

面试官通过项目经验考察候选人的**技术深度、架构能力和问题解决能力**。推荐使用 **STAR 法则**组织回答：

| STAR 要素 | 说明 | 示例 |
|-----------|------|------|
| S（Situation） | 项目背景、面临的挑战 | "项目日活 500 万，首屏加载时间超过 5 秒" |
| T（Task） | 你的任务和目标 | "需要将首屏时间优化到 2 秒以内" |
| A（Action） | 你采取的具体行动 | "通过代码分割 + 预加载关键资源 + SSR，从构建到运行时全链路优化" |
| R（Result） | 量化结果 | "首屏时间从 5.2s 降至 1.8s，LCP 提升 65%，用户留存率提升 12%" |

## 刷题优先级策略

### P0：核心（必须精通）

这些是面试中最频繁考察的内容，决定你能否进入后续面试环节：

1. **JavaScript 手写题**（一面核心）
   - 防抖（debounce）-- 必考
   - 节流（throttle）-- 必考
   - 深拷贝（deepClone）-- 必考
   - Promise.all / Promise.race -- 必考
   - bind/call/apply -- 高频
   - 函数柯里化 -- 高频

2. **框架原理**（二面核心）
   - Vue：响应式原理（Object.defineProperty vs Proxy）
   - React：Fiber 架构、Hooks 原理
   - 组件通信、生命周期、性能优化

### P1：重要（必须掌握）

3. **算法基础**（代码面）
   - LRU 缓存、二叉树遍历、三数之和、有效括号
   - 滑动窗口（最长不重复子串）
   - 动态规划入门（买卖股票）

4. **浏览器原理**
   - 渲染流程、Event Loop、HTTP 缓存
   - XSS/CSRF 防御

5. **系统设计**
   - 虚拟列表、状态管理库

### P2：加分（锦上添花）

6. **热点技术**（技术视野）
   - RSC、INP、Vapor Mode、Vite 6


## 面试回答框架：STAR-L

面试中回答问题，推荐使用扩展版 STAR-L 框架：

1. **S（Scenario）**：先给出概念定义，一句话说清楚"是什么"
2. **T（Theory）**：解释底层原理，展示深度理解
3. **A（Application）**：举出实际应用场景，证明你不仅懂理论
4. **R（Related）**：关联相关知识，展示广度
5. **L（Limitation）**：指出该技术的局限性/陷阱，展示批判性思维

**示例：面试官问"说一下 Vue3 的响应式原理"**

> "Vue3 使用 Proxy 替代 Vue2 的 Object.defineProperty 实现响应式（S）。Proxy 可以代理整个对象而非单个属性，能拦截包括属性新增/删除在内的 13 种操作，配合 Reflect 保证 this 指向正确（T）。在实践中，reactive 用于对象，ref 用于基本类型，两者在 effect 触发时通过 track 收集依赖、trigger 触发更新（A）。需要注意 Proxy 不支持 IE11，且对大对象的递归代理仍有初始化开销（L）。"

## 面试高频追问陷阱

::: danger 常见"连环追问"

**Q：说一下闭包？**
↓ 追问：闭包会造成内存泄漏吗？为什么？怎么解决？
↓ 追问：你能用闭包实现一个简单的模块模式吗？

**Q：Promise.all 有什么缺点？**
↓ 追问：那 allSettled 和 all 有什么区别？
↓ 追问：如果需要并发控制（比如 100 个请求但限制并发 5 个），你怎么实现？

**Q：Vue3 和 React 你更倾向哪个？**
↓ 追问：（选 Vue3）那 React 的 Fiber 架构解决了什么问题？
↓ 追问：（选 React）那 Vue3 的编译优化你了解多少？
↓ 追问：什么场景你会选择另一个框架？
:::

::: warning 面试策略
- 回答问题时**主动引导**到自己的优势领域（如："除了基本实现，我还可以用 WeakMap 处理循环引用问题"）
- 对于不够熟悉的问题，**诚实但不消极**（如："这个方向我经验不多，但据我了解..."然后给出你知道的内容）
- 在项目介绍中**预设亮点钩子**，引导面试官追问你准备好的内容
:::

## 面试行为问题准备

除了技术问题，大厂面试通常还会考察行为问题（Behavioral Questions）。以下是最常见的几类：

### 常见行为问题

| 类别 | 典型问题 | 回答要点 |
|------|----------|----------|
| 项目难点 | "说说你遇到过最大的技术挑战" | 用 STAR 法则，突出技术深度和解决思路 |
| 团队协作 | "与同事有技术分歧时怎么处理？" | 强调数据驱动决策，而非主观争论 |
| 技术成长 | "最近学了什么新技术？" | 展示学习能力和技术热情，说具体技术细节 |
| 失败经历 | "说说你失败的一次经历" | 重点在"从中学到了什么"和"如何改进" |
| 架构决策 | "为什么选择 A 技术而不是 B？" | 展示技术选型的权衡能力（trade-off） |

### 行为问题回答框架

```
1. 背景（Context）：简述项目背景和你的角色
2. 挑战（Challenge）：具体遇到了什么问题
3. 行动（Action）：你做了什么来解决问题（技术细节）
4. 结果（Result）：量化的成果（如有数据）
5. 反思（Reflection）：如果重来，你会怎么做
```

::: tip 行为问题加分技巧
- 使用**具体数据**而非模糊描述（"性能提升 40%"而非"性能大幅提升"）
- 展示**技术深度**（不要只说"用了 React"，要说"用了 React 18 的 Concurrent Mode 解决了..."）
- 展示**主动性**（不要等别人发现问题，而是自己发现并推动解决）
- 在回答中**预设技术钩子**，引导面试官追问你擅长的领域
:::

## 大厂面试流程解析

### 典型流程（以字节跳动/阿里/腾讯为例）

| 轮次 | 内容 | 时长 | 考察重点 |
|------|------|------|----------|
| 一面 | 基础 + 手写题 | 45~60min | JavaScript 基础、框架原理、手写代码 |
| 二面 | 项目 + 框架深度 | 45~60min | 项目经验、架构设计、技术深度 |
| 三面 | 系统设计 + 技术视野 | 45~60min | 系统设计能力、技术选型、架构思维 |
| 交叉面 | 综合能力 | 45~60min | 跨团队协作、沟通能力、技术广度 |
| HR 面 | 综合素质 | 30~45min | 职业规划、团队匹配、价值观 |

### 各轮次准备重点

**一面（技术基础）**：
- 手写题必须烂熟于心（debounce/throttle/deepClone/Promise.all）
- JavaScript 核心概念（原型链、闭包、Event Loop）能深入讲解
- 框架基本使用和原理理解

**二面（项目深度）**：
- 准备 2~3 个核心项目，用 STAR 法则包装
- 对项目中的技术选型能说出理由（为什么用这个方案而不是另一个）
- 展示性能优化和问题排查能力

**三面/交叉面（架构能力）**：
- 系统设计题的准备（虚拟列表、状态管理、路由设计）
- 技术视野：对行业趋势有自己的理解
- 能进行技术选型的 trade-off 分析

## 面试常见误区

::: danger 十大常见误区

1. **只看不写**：手写题只看代码不亲手写，面试时写不出来
2. **只背不思考**：背诵答案但不理解原理，面试官一追问就露馅
3. **过度使用术语**：为了显得专业抛出大量术语，但解释不清，适得其反
4. **项目描述假大空**：只说自己负责了什么模块，说不出具体技术细节
5. **忽略边界情况**：算法题不考虑空输入、极端值、边界条件
6. **不主动沟通**：遇到不会的题目不主动说明思路，沉默等待
7. **回答过于简短**：面试官问"了解闭包吗"，只回答"了解"，不展开解释
8. **不展示思维过程**：直接写代码不解释思路，面试官看的是思考过程
9. **技术栈单一**：只用过 Vue 但完全不了解 React（或反之），缺乏广度
10. **不准备反问环节**：面试官问"你有什么想问的"，回答"没有"
:::

## 面试反问环节

面试结束前，面试官通常会问"你有什么想问的？"。这是展示你对团队和岗位关注度的机会：

### 推荐的反问

| 方向 | 问题示例 | 目的 |
|------|----------|------|
| 技术栈 | "团队目前主要使用什么技术栈？是否有在尝试新技术？" | 了解技术环境 |
| 业务 | "这个岗位主要负责哪些业务线？" | 了解工作内容 |
| 成长 | "团队对新人的培养机制是怎样的？" | 了解成长空间 |
| 流程 | "团队的代码审查和发布流程是怎样的？" | 了解工程文化 |
| 挑战 | "团队目前面临的最大技术挑战是什么？" | 展示你能带来的价值 |

### 避免的反问

- "我通过了没有？"（太直接，面试官也需要时间评估）
- "公司福利怎么样？"（留给 HR 面问）
- "加班多吗？"（暗示你可能不愿意投入）
- 什么都不问（显得不感兴趣）
