# 面试冲刺

## 模块概述

面试冲刺模块是各技术模块的"司令部"，提供全局性的面试策略、复习路线规划、高频真题汇总和模拟面试资源。当你已经完成了各模块的专项学习后，这里将帮助你串联知识体系、补齐薄弱环节、打磨面试表现。

::: tip 核心理念
面试不是考试，而是**展示你的技术深度和解决问题的能力**。会多少不重要，能讲多少才重要。
:::

::: info 本模块定位
不重复各模块的技术细节，聚焦**面试策略**、**知识串联**、**真题训练**和**软技能**。
:::

## 核心知识点

### 面试策略

| 子模块 | 核心内容 |
|--------|----------|
| 流程全景 | 简历筛选 → 笔试（可选）→ 技术一面（基础）→ 技术二面（深度）→ 技术三面（架构）→ HR 面 |
| 准备节奏 | 基础巩固期 → 专项突破期 → 模拟冲刺期 → 复盘调整期 |

| 时间规划 | 在职准备每天 2-3 小时、全职准备每天 6-8 小时、总周期 2-3 个月 |

## 面试重点

::: warning 各轮面试侧重点
**技术一面（基础）**：Java 基础、集合、并发、JVM、MySQL 索引、Redis 数据结构。考察广度。
**技术二面（深度）**：源码阅读、线上问题排查、中间件原理、项目难点。考察深度。
**技术三面（架构）**：系统设计（秒杀/IM/推送）、分布式方案选型、容量规划、技术前瞻性。
**HR 面**：职业规划、离职原因、期望薪资、价值观匹配。
:::

::: danger 常见致命错误
- **简历造假**：项目经历夸大，被深挖时答不上来——一票否决
- **不会装懂**：遇到没准备的问题强行编造——坦诚说不会，可以补充相关知识点
- **态度消极**：抱怨前公司/前同事——面试官会担心你入职后也这样
- **准备不足**：基本概念都答错（HashMap 线程安全、String 不可变原因等）
- **没有深度**：每个问题都只答表面，无法深入——被判定为初/中级水平
:::

## 学习建议

### 阶段一：基础巩固（2-3 周）
1. 将各模块的核心知识点快速过一遍，标记薄弱环节
2. 每天刷 10 道高频面试题，用录音检验表达质量
3. 整理自己的"面试武器库"——每个知识点准备 3 层深度的回答（概念 → 原理 → 源码/实战）

### 阶段二：专项突破（2-3 周）
4. 针对薄弱模块深度学习（例如 JVM 源码、RocketMQ 存储设计）
5. 每周完成 2 个系统设计题的练习（画出完整架构图和数据流）
6. 用 STAR 法则重新梳理 3 个核心项目经历

### 阶段三：模拟冲刺（1-2 周）
7. 找朋友或同事进行 3-5 次模拟面试，全程录音
8. 分析录音中的问题：表达不清、知识盲区、逻辑跳跃
9. 针对性地补充和修正

### 阶段四：面试实战
10. 小厂练手（3-5 家）→ 中等公司过渡（3-5 家）→ 目标公司冲刺
11. 每轮面试后当晚复盘，记录不会的题目和回答不好的问题
12. 维持心态，面试有运气成分，没拿到 Offer 不代表能力不行

### 每日复习节奏（模板）

| 时间段 | 内容 |
|--------|------|
| 通勤/碎片时间 | 刷面经、听技术播客 |
