# 分布式理论

> 构建大规模互联网基础设施的基石，不仅在于"能跑起来"，更在于对分布式系统本质原理的深刻理解。

## 什么是分布式系统

**分布式系统**是指由多台独立的计算机节点通过网络互联、协同工作，对外呈现为统一整体的系统。Leslie Lamport 给出过一个经典定义："分布式系统就是这样一个系统，你甚至不知道存在一台你从未听说过的计算机的故障，就可能导致你自己的计算机无法使用。"

分布式系统的核心特征：

- **多节点协作**：每个节点拥有独立的 CPU、内存和存储；
- **通过网络通信**：节点之间通过不可靠的网络传递消息；
- **无全局时钟**：不存在一个统一的物理时钟来给所有事件排序；
- **对用户透明**：不论底层有多少节点，用户看到的应是一个完整的系统。

## 核心挑战

在分布式系统中，有八个字是绕不开的诅咒——**"网络不可靠、时钟不同步"**。

| 挑战 | 具体表现 | 导致的后果 |
|------|----------|------------|
| **网络不可靠** | 消息延迟、丢包、乱序、网络分区 | 节点间状态不一致，无法判断对方是宕机还是网络不通 |
| **时钟不同步** | 各节点物理时钟存在漂移，NTP 同步精度有限 | 事件顺序难以确定，依赖时间的逻辑可能出错 |
| **节点故障** | 进程崩溃、磁盘损坏、OOM、机器宕机 | 数据丢失、服务不可用、脑裂 |
| **数据一致性** | 多个副本之间如何保持相同的数据视图 | 读到旧数据、写入丢失、更新冲突 |

CAP 定理是分布式系统设计的"第一性原理"，揭示了**一致性（Consistency）、可用性（Availability）、分区容错性（Partition Tolerance）** 三者之间的关系。BASE 理论则是对 CAP 中 AP 方案的工程实践补充。

- CAP 定理深度解析（三要素定义、CP vs AP 本质选择）
- BASE 理论（BA / Soft State / Eventually Consistent）
- BASE vs ACID 全面对比
- 分布式一致性模型详解
- 各主流中间件的 CAP 定位映射

### 2. 一致性协议

[:arrow_right: 查看详情](./consensus.md)

共识（Consensus）是分布式系统最核心的问题：**一组节点如何就某个共同的值或决策达成一致**，即使部分节点可能出现故障。

- Paxos 算法（Basic Paxos 三角色模型、两阶段流程、Multi-Paxos 优化）
- Raft 算法（Leader 选举、日志复制、安全性三大子问题）
- ZAB 协议（ZooKeeper 原子广播协议，四模式详解）
- 其他协议（2PC / 3PC、Gossip 协议）
- 集群架构模式（主从、对等、分区与分片、故障检测）
