模块概述 ​

前端面试已经从"会用框架"时代进入"理解原理 + 系统设计 + 工程化思维"的时代。本模块为高级前端面试准备提供系统化的知识体系，涵盖手写题、框架原理、算法真题、系统设计和热点问答五大核心领域。

2025-2026 面试趋势

  • 手写题比重持续上升：大厂一面几乎必考手写题（Promise.all / 深拷贝 / 防抖节流）
  • 框架深度要求提高：不再满足于"用过"Vue3/React19，要求能说出原理和设计取舍
  • 系统设计成为高频考点：虚拟列表、状态管理库、路由系统设计
  • 算法难度向中等靠拢：链表/树/动态规划成为标配
  • TypeScript 成为默认要求：大厂前端岗位 90% 以上要求 TS 能力

前端算法特点

前端算法通常不考太难的（如红黑树、KMP），重点考察工程思维 + 算法基础的结合，如 LRU 缓存就是典型的前端算法题。

刷题优先级策略 ​

P0：核心（必须精通） ​

这些是面试中最频繁考察的内容，决定你能否进入后续面试环节：

  1. JavaScript 手写题（一面核心）

    • 防抖（debounce）-- 必考
    • 节流（throttle）-- 必考
    • 深拷贝（deepClone）-- 必考
    • Promise.all / Promise.race -- 必考
    • bind/call/apply -- 高频
    • 函数柯里化 -- 高频

  2. 框架原理（二面核心）

    • Vue：响应式原理（Object.defineProperty vs Proxy）
    • React：Fiber 架构、Hooks 原理
    • 组件通信、生命周期、性能优化

P1：重要（必须掌握） ​

  1. 算法基础（代码面）

    • LRU 缓存、二叉树遍历、三数之和、有效括号
    • 滑动窗口（最长不重复子串）
    • 动态规划入门（买卖股票）

  2. 浏览器原理

    • 渲染流程、Event Loop、HTTP 缓存
    • XSS/CSRF 防御

  3. 系统设计

    • 虚拟列表、状态管理库

P2：加分（锦上添花） ​

  1. 热点技术（技术视野）
    • RSC、INP、Vapor Mode、Vite 6

面试回答框架：STAR-L ​

面试中回答问题，推荐使用扩展版 STAR-L 框架：

  1. S（Scenario）：先给出概念定义，一句话说清楚"是什么"
  2. T（Theory）：解释底层原理，展示深度理解
  3. A（Application）：举出实际应用场景，证明你不仅懂理论
  4. R（Related）：关联相关知识，展示广度
  5. L（Limitation）：指出该技术的局限性/陷阱，展示批判性思维

示例：面试官问"说一下 Vue3 的响应式原理"

"Vue3 使用 Proxy 替代 Vue2 的 Object.defineProperty 实现响应式（S）。Proxy 可以代理整个对象而非单个属性，能拦截包括属性新增/删除在内的 13 种操作，配合 Reflect 保证 this 指向正确（T）。在实践中，reactive 用于对象，ref 用于基本类型，两者在 effect 触发时通过 track 收集依赖、trigger 触发更新（A）。需要注意 Proxy 不支持 IE11，且对大对象的递归代理仍有初始化开销（L）。"

面试高频追问陷阱 ​

常见"连环追问"

Q：说一下闭包？ ↓ 追问：闭包会造成内存泄漏吗？为什么？怎么解决？ ↓ 追问：你能用闭包实现一个简单的模块模式吗？

Q：Promise.all 有什么缺点？ ↓ 追问：那 allSettled 和 all 有什么区别？ ↓ 追问：如果需要并发控制（比如 100 个请求但限制并发 5 个），你怎么实现？

Q：Vue3 和 React 你更倾向哪个？ ↓ 追问：（选 Vue3）那 React 的 Fiber 架构解决了什么问题？ ↓ 追问：（选 React）那 Vue3 的编译优化你了解多少？ ↓ 追问：什么场景你会选择另一个框架？

面试策略

  • 回答问题时主动引导到自己的优势领域（如："除了基本实现，我还可以用 WeakMap 处理循环引用问题"）
  • 对于不够熟悉的问题，诚实但不消极（如："这个方向我经验不多，但据我了解..."然后给出你知道的内容）
  • 在项目介绍中预设亮点钩子，引导面试官追问你准备好的内容

行为问题加分技巧

  • 使用具体数据而非模糊描述（"性能提升 40%"而非"性能大幅提升"）

大厂面试流程解析 ​

典型流程（以字节跳动/阿里/腾讯为例） ​

轮次内容时长考察重点

二面项目 + 框架深度45~60min项目经验、架构设计、技术深度

各轮次准备重点 ​

一面（技术基础）：

  • 手写题必须烂熟于心（debounce/throttle/deepClone/Promise.all）
  • JavaScript 核心概念（原型链、闭包、Event Loop）能深入讲解
  • 框架基本使用和原理理解

二面（项目深度）：

  • 准备 2~3 个核心项目，用 STAR 法则包装
  • 对项目中的技术选型能说出理由（为什么用这个方案而不是另一个）
  • 展示性能优化和问题排查能力

三面/交叉面（架构能力）：

  • 系统设计题的准备（虚拟列表、状态管理、路由设计）
  • 技术视野：对行业趋势有自己的理解
  • 能进行技术选型的 trade-off 分析

面试常见误区 ​

十大常见误区

  1. 只看不写：手写题只看代码不亲手写，面试时写不出来
  2. 只背不思考：背诵答案但不理解原理，面试官一追问就露馅
  3. 过度使用术语：为了显得专业抛出大量术语，但解释不清，适得其反
  4. 项目描述假大空：只说自己负责了什么模块，说不出具体技术细节
  5. 忽略边界情况：算法题不考虑空输入、极端值、边界条件
  6. 不主动沟通：遇到不会的题目不主动说明思路，沉默等待
  7. 回答过于简短：面试官问"了解闭包吗"，只回答"了解"，不展开解释
  8. 不展示思维过程：直接写代码不解释思路，面试官看的是思考过程
  9. 技术栈单一：只用过 Vue 但完全不了解 React（或反之），缺乏广度
  10. 不准备反问环节：面试官问"你有什么想问的"，回答"没有"

面试反问环节 ​

面试结束前，面试官通常会问"你有什么想问的？"。这是展示你对团队和岗位关注度的机会：

推荐的反问 ​

方向问题示例目的
技术栈"团队目前主要使用什么技术栈？是否有在尝试新技术？"了解技术环境
业务"这个岗位主要负责哪些业务线？"了解工作内容
成长"团队对新人的培养机制是怎样的？"了解成长空间

挑战"团队目前面临的最大技术挑战是什么？"展示你能带来的价值

避免的反问 ​

  • "我通过了没有？"（太直接，面试官也需要时间评估）
  • "公司福利怎么样？"（留给 HR 面问）
  • "加班多吗？"（暗示你可能不愿意投入）
  • 什么都不问（显得不感兴趣）

面试前最后检查清单 ​
