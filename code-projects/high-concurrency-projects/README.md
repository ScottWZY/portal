### 学习目标

- 完整掌握高并发秒杀系统的架构设计与演进路径
- 理解从单体 -> 缓存 -> 消息队列 -> 微服务的架构演进过程
- 掌握秒杀系统的核心难点解决方案：超卖、黄牛、流量削峰、库存预热
- 能在压测后分析性能瓶颈并针对性优化
- 理解前端层面的优化手段（静态化、CDN、按钮防抖）

### 架构演进路径（由简到繁逐步实现）

```
V1.0: 单体架构（Spring Boot + MySQL）
  └── 基础功能：商品列表、详情、下单
      问题：直接操作数据库，并发能力差

V2.0: 缓存优化（+ Redis 缓存热点数据）
  └── 页面缓存、对象缓存
      新问题：缓存与数据库不一致

V3.0: 解决超卖（+ Redis 库存预扣 + 分布式锁）
  └── 使用 Redis 原子操作预扣库存，DB 最终扣减
      新问题：大量请求瞬间打到 Redis

V4.0: 流量削峰（+ Kafka 异步下单）
  └── 请求先入 Kafka，后端异步消费下单
      新问题：前端轮询体验差

V5.0: 完整方案（+ 限流 + 验证码 + 地址隐藏）
  └── Sentinel 限流、数学验证码、动态秒杀地址
      生产就绪版本
```

## 二、rate-limiter-demo —— 限流算法实现

### 技术栈

| 类别 | 技术 |
|------|------|
| 语言 | Java 17 |
| 框架 | Spring Boot 3.x |
| 缓存 | Redis 7.x（分布式限流） |
| 测试 | JUnit 5 + 多线程并发测试 |
| 可视化 | 自定义限流监控面板 |

### 学习目标

- 深入理解四种核心限流算法的原理和代码实现
- 掌握单机限流与分布式限流的区别和适用场景
- 能根据业务需求选择合适的限流算法
- 理解 Sentinel 的滑动窗口实现原理
- 能实现一个生产可用的限流注解

### 限流算法对比

| 算法 | 原理 | 优点 | 缺点 | 适用场景 |
|------|------|------|------|----------|
| 固定窗口 | 将时间划分为固定窗口，每个窗口有计数器 | 实现简单 | 临界突发问题 | 对精度要求不高的场景 |
| 滑动窗口 | 将窗口再细分为多个小格子，滑动统计 | 平滑、精确 | 内存占用较高 | 需要精确限流的场景 |
| 漏桶 | 请求入桶，固定速率流出 | 流量整形效果好 | 无法应对突发流量 | 需要平滑流量的场景 |
| 令牌桶 | 固定速率生成令牌，请求需获取令牌 | 可应对突发流量 | 实现相对复杂 | 允许一定突发的场景 |

### 核心实验清单

| 编号 | 实验名称 | 说明 |
|------|----------|------|
| RL-01 | 固定窗口 | 基于 AtomicLong 实现，验证临界问题 |
| RL-02 | 滑动窗口 | 基于 Redis ZSet 实现分布式滑动窗口 |
| RL-03 | 滑动窗口优化 | 基于 Array + 环形队列实现内存版滑动窗口 |
| RL-04 | 漏桶算法 | 基于 BlockingQueue 实现漏桶 |
| RL-05 | 令牌桶算法 | 基于 ScheduledThreadPool 实现令牌桶 |
| RL-06 | 令牌桶优化 | 基于 Guava RateLimiter 源码分析 |
| RL-07 | 分布式限流 | 基于 Redis + Lua 实现分布式令牌桶 |
| RL-08 | 限流注解 | 自定义 @RateLimit 注解 + AOP 拦截 |
| RL-09 | 压测对比 | 四种算法在相同条件下的性能与精度对比 |
